# DOCTRINE ENGINE — 汎用マーケ意思決定エンジン

> Status: v0 (2026-06-15)。これは「思想家ごとのルール」ではなく、
> **どんなサービスも直面する普遍的な意思決定の背骨**。サービス非依存＝汎用知能の核。

## なぜこの形か
サービス固有のルールを書くと汎用化が壊れる（=憲法違反）。
代わりに「全事業に共通する意思決定」を定義し、各決定で **どの思想が・どの状況で発火するか**
＋**対立時の裁定**を持たせる。未知のサービスも同じ背骨を流れ、AIが自動生成プロファイルで答える。

## 意思決定の背骨（The Decision Spine）
どんなサービスも、この7つの問いに順に答える。各問いに「貢献する思想家」と「対立」がある。

| # | 普遍的な問い | 主な思想家（対立を含む） |
|---|---|---|
| D1 | 我々は本当は何屋か？（ジョブ/カテゴリ定義） | レビット, ドラッカー |
| D2 | 誰のためで、どこまで広げるか？ | シャープ(広く/浸透) ⇔ ゴーディン(狭く/尖る) |
| D3 | 尖った角度は何か？（差別化の刃） | ゴーディン(パープルカウ), ジョブズ(ビジョン) |
| D4 | 需要は「刈る」か「創る」か？ | ホプキンス/オグルヴィ(刈る・DR) ⇔ マスク/ジョブズ(創る) |
| D5 | どのチャネルで何を語るか？ | ヴェイナチャック(媒体適応), オグルヴィ(クラフト) |
| D6 | どう測り、どう意思決定するか？ | シャープ(エビデンス), ホプキンス(全数テスト), ジーマン(ROI) |
| D7 | どう賭け、どう拡大するか？ | 孫(タイミング/群), マスク(第一原理) |

## 各決定の記述フォーマット（D1〜D7すべてこの形で蒸留する）
```
## D{n}: {普遍的な問い}
- 貢献思想家とポジション（対立を明示）
- 発火条件: ServiceProfile のどのシグナルで、どのポジションが選ばれるか
    （例: 市場成熟度 / ブランド段階 / 予算 / データ可用性 / カテゴリ新規性）
- 裁定方式: 1つ選ぶ or 競合2案を立てて実データで決める（既定=後者）
- 出力: この決定がワークフロー/戦略に与える具体的指示
```

## 裁定の既定方針（要・最終承認）
**「対立は無理に解消せず、競合する2案を仮説として立て、実データ(ROAS/CVR等)で決める」**を既定とする。
理由: これが最も尖り、かつ汎用（サービスごとに正解が違うため、コードで決め打ちしない）。
予算・リスク制約が厳しい時のみ、ドクトリンで1案に絞る。

# 蒸留サンプル（パターン提示）

## D1: 我々は本当は何屋か？
- **レビット**: 顧客はドリルでなく「穴」を買う。製品ではなく**達成したいジョブ**で事業を定義せよ（マーケティング近視眼を避ける）。
- **ドラッカー**: 事業の目的は顧客の創造。「我々の顧客は誰で、彼にとっての価値は何か」を問え。販売を不要にするのが理想。
- **対立**: ほぼ補完関係（珍しく協調）。ただしレビットは「広げすぎ」の罠あり（鉄道→輸送業のように広げすぎると焦点喪失）。
- **発火条件**: 全サービスで最初に必ず実行（D1はスキップ不可）。
    - ProfileにカテゴリのみでジョブTが不明 → AIが「このサービスが解く顧客のジョブ」を調査・言語化。
- **裁定**: 競合不要。ただし「狭いジョブ定義」と「広いジョブ定義」の2解像度を出し、D2以降で使い分ける。
- **出力例(B-Ticket)**: ❌「クーポンアプリ」 → ⭕「PHの生活者が"今日の出費をお得に賢く決める"不安と高揚を解決する」
    （※これはAIが調査して導く想定。手書きの固定値ではない）

## D2: 誰のためで、どこまで広げるか？
- **バイロン・シャープ**: 成長は浸透率(penetration)で決まる。ヘビーユーザー神話より**ライトユーザーの母数**。広くリーチし、メンタル/フィジカルアベイラビリティを最大化。識別資産(distinctive assets)を一貫使用。絞りすぎ厳禁。
- **セス・ゴーディン**: マスは死んだ。「全員」狙いは誰にも刺さらない。**最小実行可能オーディエンス**に尖って刺し、彼らが広める。
- **対立**: 真っ向（広く到達 ⇔ 狭く尖る）。時系列の遷移でもある（ゴーディンで火種→シャープで拡大）。
- **発火条件**: 無名/初期・予算小 → ゴーディン。カテゴリ確立・到達原資あり・浸透が成長ドライバー → シャープ。
- **裁定**: 既定＝競合2案（狭×高熱量 / 広×高到達）を別キャンペーンでテスト、CAC/CVR/浸透で決定。
- **出力例**: B-Ticket初期 → ゴーディン的に熱量セグメントへ尖って投下、データが出たらシャープ的拡大へ遷移。

## D3: 尖った角度は何か？（差別化の刃）
- **ゴーディン（パープルカウ）**: 退屈が最大のリスク。製品/メッセージに「語らずにいられない異質さ」を埋める。
- **ジョブズ**: 調査で平均を取るな。ビジョンで「顧客がまだ言葉にできない欲求」を提示。**引き算**（削る勇気）。体験全体を磨く。
- **オグルヴィ（拘束条件）**: ただし"We sell, or else"。**売れる尖り**であれ。アートのための尖りは禁止。
- **対立**: ゴーディン(外向き話題性) ⇔ ジョブズ(内発ビジョン＋引き算)。両者とも「平均化＝死」で一致（＝憲法と同義）。
- **発火条件**: 製品に構造的独自性あり → ジョブズ（製品=広告）。コモディティ寄りで勝負がメッセージ → ゴーディン。
- **裁定**: これが **BoringFilter の基準そのもの**。「競合の誰でも言えるか／スクロールを止めるか」を通らなければ却下。複数角度生成→Filter→上位をテスト。
- **出力**: 各サービスで「**捨てるもの**」を必ず1つ決める。全部入りは尖らない。

## D4: 需要を刈るか創るか？
- **ホプキンス/オグルヴィ（刈る・DR）**: 既に欲しい人を捕える。検索/リスティング/リターゲ。全数テスト・全数トラッキング、reason why、即行動。
- **マスク/ジョブズ（創る）**: まだ存在しない需要を製品とビジョンで生む。第一原理でカテゴリ再定義。広告より製品が語る。
- **対立**: 既存需要の刈取り ⇔ 新規需要の創造。チャネルも指標も別（CPA ⇔ 認知/態度変容/PR）。
- **発火条件**: 検索ボリューム・カテゴリ認知あり → 刈る。新カテゴリ/行動変容が必要・検索が無い → 創る。
- **裁定**: 既定＝下部ファネル(刈取り)で即ROI確保しつつ、上部(創出)を仮説検証。比率はデータで動的に。
- **出力**: ファネル別予算配分＋各層の指標。

## D5: どのチャネルで何を語るか？
- **ヴェイナチャック**: 各媒体のネイティブ文脈に合わせて作れ（使い回し＝死）。物量と注意経済。ジャブ→右フック（与えてから求める）。
- **オグルヴィ**: 媒体が変わってもクラフト原則は不変。**見出しで8割**決まる。売れるなら長文も可。
- **対立**: 補完だが緊張（媒体適応・量 ⇔ 普遍クラフト・質）。
- **発火条件**: チャネルはD2/D4の結論＋Profileの有効チャネル(Meta/リスティング/SNS/LP/SMS)から選定。各チャネルにヴェイナ的ネイティブ化＋オグルヴィ的クラフトを**両方**適用。
- **裁定**: チャネルごとにメッセージ別生成（使い回し禁止）。各チャネルでBoringFilter通過必須。
- **出力**: チャネル×メッセージのマトリクス。SMS(PH文脈)は簡潔・即時性・オファー明確を最優先。

## D6: どう測り、どう意思決定するか？
- **シャープ**: 虚栄指標でなく浸透率・到達・識別資産の一貫性。短期刈取り偏重は長期ブランドを毀損。
- **ホプキンス**: すべてテスト・トラッキング。推測排除。クーポン/コードで反応計測（B-Ticketと相性◎）。
- **ジーマン**: マーケはROIで説明責任。売上に繋がらない活動を切れ。
- **対立**: シャープ(長期ブランド) ⇔ ホプキンス/ジーマン(短期・直接反応・ROI)。
- **発火条件**: 全サービス必須。短期KPI(ROAS/CPA/CVR)＋長期KPI(到達/想起/浸透)の**両建て**を既定。データ可用性が低ければ先に計測基盤(イベント設計)を立てる。
- **裁定**: 二重スコア（短期効率＋長期健全性）で意思決定。短期のみで回すと憲法違反（凡庸な刈取り機）に堕ちる。
- **出力**: 指標定義＋意思決定ルール（どの指標がどの閾値で次アクション）。

## D7: どう賭け、どう拡大するか？
- **孫正義**: タイムマシン経営（先行市場の型を持込む）、群戦略（複数の賭けを束ねる）、勝てる所に大胆集中。タイミングを読む。
- **マスク**: 第一原理（慣習でなく本質から再構築）。コスト構造を根本から壊す。指数成長は製品・体験・口コミで。
- **対立**: 孫(市場/タイミング/資本の大局) ⇔ マスク(製品/コスト/第一原理)。
- **発火条件**: スケール判断時に発火。勝ちパターンが実データで確認された施策に孫的に集中投下。単位経済が破綻 → マスク的に**再設計**（チューニングでなく構造作り直し）。
- **裁定**: 勝ち筋が実データで確認された後のみ大胆投下（それ以前の拡大は禁止＝slop拡散）。
- **出力**: スケール条件（指標がN水準超で予算M倍）＋撤退条件。

# 実行時の流れ（このエンジンがどう動くか）
```
新サービス登録（管理画面・最小起点のみ）
  ↓
DISCOVER: AIが対象を自動調査 → ServiceProfile 生成（固定値なし）
  ↓
Doctrine Engine が D1→D7 を走査（各決定で発火思想を選ぶ／対立は競合仮説に）
  ↓
Workflow Planner: 上記の結論からこのサービス最適の手順を動的に組む
  ↓
生成（思想に振り切る）→ BoringFilter（D3基準で凡庸を却下）
  ↓
人間の承認ゲート（実課金前・RULES第4条）
  ↓
ChannelAdapter で配信 → 実績取込
  ↓
D6 の二重スコアで評価 → memory(pgvector)に学習 → 次サイクルへ改善
```
サービス固有のコードは一切無い。違いは全て「AIが導いたProfile＋D1〜D7の結論」に宿る。
